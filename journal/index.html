Week 1/Week 2 Introduction and HTML

Summary of New Skills

The unit started with an introduction to the web and HTML, learning about some of its history and its significance in regard to the modern internet. I learned what a markup language is and how it is distinct from a programming language, and how browsers parse markup to render content in a browser window.

We looked at the basic anatomy (syntax) of HTML elements and more specifically at HTML5 semantic elements such as <header>, <nav>, <main>, and <footer> discovering their functions and relevance. We also explored attributes and why they might be necessary, and how an HTML document should be structured overall.

Additionally, I learnt about basic form elements (e.g., <input>, <select>, <option>) and the required attribute for simple form validation. The tutorial task included creating multiple pages (index.html and registration.html), linking them, and ensuring that both share consistent headers and footers.

Tutorial Task and Self-Assessment

For the tutorial, I completed two basic HTML pages, index.html and registration.html. Both pages were structured using HTML elements and included the same headers and footers across both. The pages were linked for navigation, and the registration page included a form that had input fields, selections, checkboxes, and validation attributes. I met some minor difficulties, such as attribute typos and validation errors, but these issues were fairly easily resolved through checking and corrections.

After completing this week's content and tutorial activities, I feel more confident in my understanding of HTML and ability to structure HTML pages semantically. My understanding of form elements and attributes has significantly improved. However, I sometimes remain uncertain about the correct placement of some attributes and the semantic relationships between elements, this can often be overcome by revisiting the content or googling my issue, besides this there was little else that I struggled with this week. Overall, my confidence with HTML has increased, and I feel comfortable creating basic HTML documents independently.

Week 3 CSS

Summary of New Skills

This week approached the concept of CSS (Cascading Style Sheets), which I learnt is used to style and visually format webpages. It was explained that CCS is only responsible for these visual elements and therefore must be paired with a markup language file such as HTML. I discovered how CSS changes elements in through selectors, including selecting by type, class, and/or ID. We were also taught about more advanced selectors such as attribute selectors, pseudo-classes, and pseudo-elements. Combinators were introduced, which select elements based on their relationship in the HTML structure. Fundamental CSS properties like fonts, colours, and margins were covered as well. The importance of organising CSS clearly for readability and maintainability was highlighted.

Tutorial Task

This week's tutorial involved applying the new CSS techniques to style the HTML pages created previously. The styling undertaken included centring page content within a container, adding background colours, and styling page elements such as headings, links, and buttons. and structuring the registration form neatly into two columns using CSS Grid allowing the labels and input fields to be better aligned. I encountered several issues during the process, particularly with the grid structure on the registration page, but these were mostly resolved through troubleshooting.

Self-Assessment

I feel I've learnt a lot from this week's content, particularly through the tutorial activity. I found the CSS concepts presented in the lectures and self-study materials to be confusing, as they felt quite different from the programming concepts I'm familiar with. However, actively working through the tutorial tasks made everything feel clearer, and it all began to fall into place. My confidence in applying CSS has significantly improved and I feel comfortable styling HTML element independently, although there are still some aspects that challenge me.

What Was Most Confusing

I particularly struggled with aspects of the CSS Grid layout. During the tutorial, I couldn’t get the columns to arrange and lay out in the way that I intended. Through trial and error, I discovered there are different justify-* properties, and using justify-content instead of justify-items gave me a result closest to what I wanted. Despite spending time trying to understand it, I still don’t fully get why this made such a difference. I also have no idea about more advanced pseudo-classes and especially pseudo-elements, although these weren't required for the tutorial. Maybe these will be revisited in the future.

Week 4 UX & Accessibility

Summary of New Skills

This week further built on my knowledge of CSS by introducing responsive design, specifically learning about media queries and the Flexbox layout system. This taught me ways to make pages adapt to different screen sizes by adjusting layouts and style, making websites more usable on mobile devices. In addition, User Experience (UX) principles were introduced focusing on design, navigation, and hierarchy. Accessibility (A11Y) was also introduced in which we learnt about practices such as using semantic HTML, colour contrast principles, and adding labels to improve the experience for less able users.

Tutorial Task

The tutorial task looked at making the marathon website more responsive and fit better on smaller screens (such as mobiles). Both the current pages were updated to include the viewport meta tag as well as having media queries added to adjust styles based on screen size. I also restructured the registration form layout for small screens by modifying the existing CSS Grid setup. I created a new testimonials page which included Flexbox to arrange the content in a flexible and hopefully responsive layout. Overall, this activity provided an opportunity to practise responsive design techniques and highlighted the importance of accessibility in layout design.

Self-Assessment and Reflection

I found this week’s content relatively easy to understand and straightforward. Perhaps worth mentioning that adding the viewport meta tag and working with media queries was conceptually new to me and was a little confusing to begin with but after experimenting a little bit it made sense quickly. Using Flexbox to structure the testimonials page was simple and intuitive after carefully reading the task instructions as was restructuring the CSS Grid for smaller screen. It felt that this week was more about reinforcing existing concepts rather than approaching anything particularly new and difficult.

Week 5 Introducing JavaScript

Summary of New Skills

This week’s content introduces JavaScript and the third core web technology taught in the unit. I learned how to allow a script to interact with a webpage using the <script> tag and the importance of the defer attribute in forcing scripts to run after the page has loaded, and why this is important. Basic JavaScript syntax was introduced, including declaring variables, common data types, and control structures. I also learned how to use template literals to simplify string formatting. Functions and scoping rules were introduced such as hoisting and the differences between global, function, and block scope. Finally, basic debugging tools were mentioned along with the browser's developer tools to test and debug scripts.

Tutorial Task

The tutorial involved several small JavaScript activities. I connected a script file to an HTML page and used consol.log() to test functionality. The first activity involved calculating averages and comparing values using functions and conditional statements. I used for loops to iterate through arrays and objects which demonstrated the different types of data structures in JavaScript. There was the tax calculator task, this involved using conditionals to apply different tax brackets and template literals to format string outputs. I also wrote a function that injects content into the webpage using innerHTML, showing how JavaScript interacts with the DOM. All tasks helped me to better understand how JavaScript works alongside HTML.

Self-Assessment and Reflection

This week’s content was quite engaging and felt more familiar to me than other weeks. Given previous experience with programming units and languages, JavaScript feels more like what I’m used to, given it’s a true programming language rather than a markup or styling language like the others so far in the unit. The syntax, use of variables, functions, and control structures all felt intuitive and similar to concepts I’ve seen before. Learning about functions to process data and interact with the page was useful as it helped me understand how logic and layout can come together. I did find some parts like scoping rules and loop variations tricky to understand at first, but they quickly made more sense after using them in the tutorial. At this stage I’m feeling confident using JavaScript for page interaction and logic and feel comfortable with building on this in the coming weeks.

Week 6 The Document Object Model

Summary of New Skills

This week introduced the Document Object Model (DOM) which presents a webpage as a tree of elements that scripts, in this case JavaScript, can interact with. I learned how to use DOM methods like getElementByID, or querySelector, to select and change HTML elements. Events were also introduced, specifically how to use addEventListener to respond to user actions. The content also demonstrated basics of event propagation and how JavaScript can be used to validate forms dynamically.

Tutorial Task

The tutorial focused on using JavaScript to interact with the DOM to implement form validation. I removed the default HTML5 validation and implemented a validateForm() function instead. This function checked for required fields, compared password values, and showed error messages where relevant next to each input field. This approach used addEventListener to listen for the form submission and then dynamically modify the HTML page based on the received values. I used classList for applying styling from the CSS file which kept the presentation of the page separate from the validation logic. This task was a demonstration of how JavaScript can be utilised to control webpage behaviour and logic in a more flexible way than built-in methods.

Self-Assessment and Reflection

I initially found the content this week quite understandable and approachable, but the tutorial task presented a few challenges. I began the tutorial by trying to use setCustomValidity() and reportValidity() to display validation error, but after discussing with my tutor, I learned that while this method did indeed validate the fields correctly, the error messages didn’t present in a satisfactory way. Instead, I changed my approach to displaying error messages next to each form field manually using added <span> elements. This made the validation presentation clearer but required a restructuring of the JavaScript. I also had some issue with grid layout but the fix for this was simple and resulted in furthering my knowledge of the grid system. As a result of these setbacks, however, I feel more confident and believe my understanding of JavaScript and DOM manipulation has improved. I also have a greater appreciation of the importance of separating presentation from logic.

Week 8 Data Storage

Summary of new skills

This week focused on how web applications store and manage data using data bases, it particularly focused on the relation model. We covered how to design and interpret Entity Relationship Diagrams (ERDs), convert ERDs into data dictionaries, primary and foreign keys, and how they define relationships between tables. This has been covered fairly extensively in previous units, so there wasn’t much new to learn here, but it was a useful refresher and an opportunity to apply these concepts in a real-world scenario.

Tutorial Task

The tutorial involved designing a data base for the marathon website. I started by creating an ERD showing the employees, races, and participant entities, as well as an associative entity to track participants in races. Using the ERD I then created a data dictionary with defined attribute types and constraints. I think implemented the database design using phpMyAdmin and build to actual database. Creating the tables was simple but I had some trouble with adding the foreign key constraints to the associative entity. I eventually figured it out and inspected the result in the Designer view.

Self-Assessment and Reflection

This new knowledge clearly is intended to build on the earlier work with forms. Designing the ERD felt familiar and logical, and building the table in phpMyAdmin wasn’t particularly difficult other than some minor issues that were easily overcome. These issues mostly came from my unfamiliarity with the interface as most of my previous work with phpMyAdmin was directly writing SQL and importing it. This week served as a good refresher or planning and implementing databases from scratch, a skill I can see being useful in many facets of ICT.

Week 9 PHP

Summary of New Skills

This week introduced PHP, a server-side language for dynamic web content. I learnt about how PHP can be embedded within HTML using <?php ?> tags and how the server executes PHP before sending the resulting HTML to the browser. Basic syntax, carriable, control structures, and how it handles arrays and associative arrays was covered. Differences from JavaScript, like type juggling and scoping, were also discussed. Common PHP usage patterns including string formatting with echo, basic form handling, and accessing $_GET and $_POST variables where introduced.

Tutorial Task

The tutorial involved modifying a .php page that processed and displayed match data and tax calculations. I made PHP functions to calculate averages, count wins from arrays, and calculate taxes. Values were printed dynamically using echo within the HTML page. The tax calculator used associative arrays and formatted string values for clean output. The tutorial clarified how server-side code can be used to achieve logic driven pages that respond to data inputs.

Self-Assessment and Reflection

I found the content this week reasonably straightforward, especially because PHP shares a lot of syntax with other languages I’m familiar with. I did encounter the usual small issues like variable name typos and forgetting semicolons, which I noticed was particularly hard to spot in PHP for some reason. The immediate feedback when running the PHP in browser was appreciated, and I found the tutorial a helpful overall intro into basic server-side logic. Overall, I’m feeling more confident about using PHP to process data and generate dynamic output. Exploring the more advanced features in coming weeks should be interesting.

Week 11 Security

Summary of New Skills

This week focused on implementing dynamic forms using PHP and the MySQL database. I was taught two common methods for handling user input: a two-file approach that includes an HTML and PHP file, and the self-posting PHP form, which is a single PHP file. I saw how insert and update queries can be used within PHP to update tables in the database and how to use HTML5 validation to ensure fields were completed correctly. The tutorial also introduced basic security practices like sanitising output using htmlspecialchars() to prevent cross-site scripting.

Tutorial Task

The tutorial involved updating the add.html form to include validation and post to add.php. In add.php, I wrote a statement to insert a new participant record into the database and then redirect back to participants.php if successful. I completed edit.php by writing a retrieve() function that will populate the form with the existing participant's data, and an update() one to commit the changes. I tested the forms to confirm participants could be added and edited successfully.

Self-Assessment and Reflection

Introducing dynamic user input felt like it built well on last week's content. The two-file approach in the tutorial for adding records was easy enough, but the edit page stood out to me more, seeing how one file can handle both displaying and updating a record depending on whether it's accessed via GET or POST. I also ran into a few minor issues like mistyped column names and simple CSS issues, which serve as my daily reminder that any tiny oversight can break functionality. I wouldn’t say the form logic was overly difficult, but I found it did reinforce the importance of sanitising data and the use of prepared statements. Overall, it feels like a complete ‘start to end’ proof from user input to database storage and retrieval.

Week 12 Authentication

Summary of New Skills

This week, I learned how to implement basic authentication and authorisation in PHP. I learned how we can use PHP sessions to persist login state across multiple pages and how we can verify a user’s identity through secure login forms. The lecture emphasised the ‘stateless’ nature of HTTP, the use of sessions to store state data, like logged-in users, and the importance of password hashing. PHP’s password_hash() and password_verify() functions were presented as secure tools to use when storing and validating passwords. The distinction between authentication (verification of identity) and authorisation (controlling access) was also covered.

Tutorial Task

The tutorial task went over implementing login and logout functionality. Using session_start() and $_SESSION to manage the user session ensured protected pages could only be accessed by a logged-in user. The login form was created as a self-posting PHP page and initially allowed any credentials to be entered. Later, I added a User table to the database that stores hashed passwords with password_hash() and verified them using password_verify() inside a function called authenticate(). I also made a form to register new users and implemented access control with session checks on the protected pages.

Self-Assessment and Reflection

This week's content was the final part tying together frontend forms, backend logic, and database operations. It increased my understanding of how authentication can be implemented on the web, in particular the importance of hashing and the role of sessions in state management. Having to add actual authorisation checks (redirection of unauthorised users) made the site feel like a real admin system. Overall, I feel I now have a clearer picture of secure user management in PHP.
